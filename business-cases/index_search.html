Data Structures Used

Core data structures that power our efficient search ranking system

üìä

Vector Space Model

High-dimensional vector representation of documents and queries using BERT embeddings. Each document is represented as a 768-dimensional vector, enabling efficient similarity computations and semantic understanding.

Implementation Details:

  • 768-dimensional float vectors
  • Normalized for cosine similarity
  • Memory-efficient storage
  • Fast vector operations
üå≥

KD-Tree

Spatial data structure for efficient nearest neighbor search in high-dimensional spaces. Used for quick retrieval of similar documents and intent clustering.

Implementation Details:

  • Balanced tree structure
  • O(log n) search complexity
  • Optimized for high dimensions
  • Efficient range queries
üìù

Priority Queue

Binary heap implementation for maintaining top-k results during submodular optimization. Ensures efficient selection of diverse and relevant documents.

Implementation Details:

  • Binary heap structure
  • O(log n) insert/delete
  • O(1) max/min access
  • Thread-safe operations
üîç

Inverted Index

Efficient document retrieval structure mapping terms to document IDs. Used for initial candidate selection before semantic processing.

Implementation Details:

  • Hash table based
  • Posting lists compression
  • Fast term lookup
  • Memory-efficient storage

Working Comparison

Traditional Search Ranking

1

Initial Document Retrieval

Search engine retrieves documents based on keyword matching or pretrained relevance scores (e.g., BM25, neural rankers).

2

Relevance Scoring

Each result is scored based on how closely it matches the query, often ignoring the diversity of content. E.g., beginner-level tutorials might dominate the top 10 results if they score highest.

3

Top-N Selection

The system selects the top 10 results with the highest scores, regardless of content overlap.

User sees results like:

  • 6 beginner tutorials
  • 3 generic tutorials
  • 1 data science tutorial
  • (Web dev or advanced topics may be missing entirely)

IndexWise System

1

Initial Document Retrieval

Retrieves the same pool of top-N results from the standard search backend.

2

Document Embedding

Each result is transformed into a semantic vector using BERT or SBERT to capture its actual meaning.

3

Intent Clustering

Results are grouped into sub-intents such as:

  • Beginner tutorials
  • Data science with Python
  • Web development using Python
  • Automation scripts, etc.

4

Relevance Scoring

Results are scored based on:

  • How relevant they are to the main query
  • How well they represent their respective sub-intent

5

Submodular Optimization

Applies a greedy selection that:

  • Chooses diverse results from different intent clusters
  • Avoids repeating the same sub-intent

User sees results like:

  • 1 beginner tutorial
  • 1 data science tutorial
  • 1 web dev tutorial
  • 1 automation script
  • 1 advanced/intermediate-level content
  • (Balanced and useful across user needs)
