🚨 Business Case & Problem Statement

Current Problem

Manipulated app reviews generated by botnets and fake accounts are used to artificially boost or demote apps on Google Play. These reviews are coordinated, happen in bursts, and often involve shared bot accounts across apps.

Current Weakness

Google's ML-based filters handle individual review authenticity but miss coordinated attacks spread across multiple apps/accounts — especially when bots imitate human-like variation.

Business Need

Build an algorithmic detection layer that flags abnormal temporal bursts, colluding app-reviewer clusters, and duplicate reviewer behavior.

🧠 Algorithm Implementation & Data Structures

Component Algorithm Purpose Data Structure
User-Review Matching XOR Fingerprinting Each user account is assigned a compact binary signature by applying XOR operations over the app IDs and timestamps it has interacted with. This helps us quickly detect bot accounts reused across apps with near-identical review activity. These signatures are stored and compared in constant time. Hash Tables
Review Timeline MO's Algorithm For each app, all reviews are sorted chronologically. MO's algorithm is then used to process batch queries that ask: "How many new reviewers appeared between time t1 and t2?" or "What's the peak review frequency in a window of 1 hour?" This highlights temporal burst patterns characteristic of fake review campaigns. Arrays + Query Blocks
App–Reviewer Graph Min-Cut / Max-Flow A bipartite graph is constructed where left nodes are users and right nodes are apps. An edge connects a user to every app they've reviewed. Min-Cut / Max-Flow separates suspicious dense clusters (e.g., 10 users reviewing 10 apps) that wouldn't exist naturally. These clusters are flagged for manual audit or auto-removal. Adjacency Lists + Flow Network

🧱 Data Structures Used

Hash Tables

Used to map each user ID to its XOR-generated fingerprint. Allows O(1) time checks for whether a new reviewer's behavior matches previously known bots.

Adjacency Lists

Stores the bipartite graph between reviewers and apps. Adjacency list is space-efficient for sparse graphs (most users review only a few apps) and supports quick traversal for the max-flow algorithm.

Arrays + Query Blocks

MO's algorithm divides the timeline into blocks (√N size) and stores the count of unique reviewers per block. Arrays track review frequency over time; helps process multiple burst-detection queries efficiently.

Flow Network

Built with a source, sink, reviewer nodes, and app nodes. Capacities are defined (e.g., how many apps a bot typically reviews). Max-flow helps separate well-connected suspicious subgraphs from the rest of the network.

🔍 Key Insights & Inferences

XOR Hashing Benefits

Tracks reused bots even when behavior is varied, providing a compact and efficient fingerprinting mechanism.

Temporal Pattern Detection

MO's algorithm detects non-random patterns in review timing — a strong indicator of automated manipulation.

Collusion Network Exposure

Min-Cut graph separation exposes clusters of cooperating malicious apps and reviewers effectively.

🏁 Solution Summary

This solution adds a scalable, algorithm-first layer to detect coordinated review manipulation in Google Play.

It complements existing ML tools and strengthens trust and ranking fairness, enabling early detection and reducing user harm and reputational risk.
